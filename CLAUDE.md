# CLAUDE.md — セミナー会員サイト プロジェクト

このファイルは Claude Code が毎回読み込むプロジェクトの前提です。会話の冒頭で必ず参照してください。

## プロジェクト概要

歯科衛生士業界のセミナー講師（石原先生）向けに、セミナー動画を「受講した人だけが見られる」会員サイトを構築する。最大の課題は、現状 YouTube/Vimeo の限定公開URLが受講者以外に拡散され、無料で視聴されてしまっていること。これを技術的に防ぐことが主目的。

このリポジトリは現在、先生に構成を説明・選定してもらうための「仕様コンフィギュレーター（単一HTMLツール）」を含む。今後、実サイト（WordPress）の構築に発展させる可能性がある。

## 絶対の前提（仕様の核）

- 会員登録は「無料」。受講（セミナー参加）で課金が発生する。会員であること自体に料金は発生しない。
- 会員ランク（有料会員/無料会員などの階層）は持たない。
- 受講者は「自分が受講したセミナーの動画だけ」をマイページで閲覧できる（受講していないものは見えない）。
- セミナーごとにパスコードを配る方式にはしない。会員は1回ログインするだけ。

## 構成は2モードに集約される

- モードX（フル）= 案内＋申込＋決済＋動画。参加者リスト・入金管理・メッセージ送信を含む。
- モードY（動画のみ）= 案内も申込もせず、受講者を登録して動画だけ配信。

## 技術スタックの方針

- 土台: WordPress（既存 Xserver 流用可）＋ LMS ＋ 外部動画ホスティング。プランを上げても土台は載せ替えない（出戻り防止）。
- LMS: Tutor LMS（無料版で開始可、必要に応じ Pro）。規模次第で LearnDash も検討。
- 動画ホスティング: Bunny Stream を第一候補（従量課金・署名URLトークン・総再生数の分析を持つ）。Vimeo は2025年末の買収後に料金/ポリシーが不安定なため第一候補にしない。
- 拡散防止: 「WPログイン」だけでは不十分。動画URLはDOMから抜けるため、必ず動画ホスティング側のドメイン制限／署名URLで塞ぐ。
- 拡散防止レベルは「弱（ログインのみ）／中（サイト外で再生不可）」の2段階。視聴者名の焼き込み（ウォーターマーク）は今回は採用しない。

## 確定した要件と充足方法

- 受講動画の紐付け: LMSのコース割り当て（標準）。
- 動画ごとの閲覧期限: 「動画単位」で設定。LMSのアクセス期限＋署名URLのTTL。受講者ごとの個別期限ではない。
- 視聴回数: 「動画単位の総再生回数」でよい（ユーザー別の個別集計は不要＝重いカスタム実装を回避）。Bunny の標準分析で対応。
- ユニーク再生: 「出れば嬉しい」程度。Bunny のセッション基準の概算で可。厳密な会員紐付けユニークは対象外。
- 入金状態管理（モードXのみ）: 入金フラグのカスタムフィールド（軽微な追加）。
- 参加者へのメッセージ（モードXのみ）: メール配信プラグイン＋イベント単位の絞り込み送信（軽い作り込み）。
- CSV出力・受講履歴・参加者リスト: LMS/会員プラグインの標準機能。

## コーディング/運用の約束

- 仕様ツール（index.html）は外部CDN依存なしの単一ファイルを維持する（GitHub Pages でそのまま動く前提）。
- ブラウザストレージ（localStorage 等）は使わない。状態は JS 変数で保持。
- 日本語でやり取りする。GUIベースのワークフローを好む（ターミナルよりアプリ/Finder/ブラウザ）。
- 変更したら、何をなぜ変えたかを簡潔に説明する。
